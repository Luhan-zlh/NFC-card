# 给凯玟的卡片 — 项目说明

纯静态网页（HTML/CSS/JS，无后端），碰 NFC 卡片后自动打开。托管在 GitHub Pages。

**当前网址**：https://luhan-zlh.github.io/NFC-card/
（这就是最终要写入 NFC 芯片的链接）

---

## 文件结构

```
nfc_redirect_site/
├── index.html          网页结构
├── manifest.json        PWA 配置（支持"添加到主屏幕"）
├── sw.js                 离线缓存脚本
├── css/style.css         深色星空视觉风格
├── js/data.js            【你唯一需要改的文件】名字/日期/时间线/小纸条/留言本链接
├── js/script.js          逻辑代码（一般不用改）
├── icons/                PWA 图标
├── SHORTCUTS_GUIDE.md    可选：给她的 iOS 快捷指令自动化教程
└── README.md             本文件
```

## 第一步：改内容

打开 `js/data.js`，按提示替换：

1. `herName`、`anniversaryDate` — 已填好（凯玟 / 2025-08-04）
2. `timeline` 数组 — 把占位的几条换成真实的重要日期和故事，想加几条都行
3. `loveNotes` 数组 — 先用占位的几句，之后随时替换成自己写的话
4. `sharedNotebookUrl` — 见下面"共享留言本"部分

照片放进 `images/` 文件夹（建议压缩到 500KB 以内），然后在时间线对应条目把 `img: ""` 改成 `img: "images/文件名.jpg"`。

---

## 第二步：共享留言本（腾讯文档，双方都能写）

1. 打开 https://docs.qq.com ，登录
2. 新建一个"在线文档"
3. 右上角"分享" → 设置为"**互联网上获得链接的人可编辑**"
4. 把生成的链接填进 `data.js` 的 `sharedNotebookUrl`

---

## 第三步：部署更新

内容改完后，需要重新上传到 GitHub 才会生效（当前由 AI 助手通过 GitHub API 上传，因为本机到 `github.com` 的直连线路不稳定，走的是 `api.github.com`，不受影响）。

如果你自己想更新，也可以正常用 git（如果本机连 github.com 网页很卡属正常现象，用 SSH 方式通常不受影响）：

```powershell
git add .
git commit -m "update content"
git push
```

**GitHub Pages 缓存提醒**：内容更新后大约 **10 分钟** 生效，测试时可以在链接后加 `?v=123` 之类的参数绕过浏览器缓存，强制刷新。

## 第四步：把链接写入 NFC 卡片

1. 手机（iOS 或 Android 均可）下载 **"NFC Tools"**（免费）
2. 打开 App → "写入" → "添加一条记录" → 选择 "URL/URI"
3. 输入：`https://luhan-zlh.github.io/NFC-card/`
4. 点击"写入" → 把手机贴近卡片背面 NFC 芯片位置 → 完成

**重要提醒**：
- 网址必须是 `https://` 开头（已经是），否则 iPhone 不会自动弹出识别横幅
- 建议先写入一次，用另一部手机测试识别效果，确认无误后再进行下面的"只读锁定"

### 4.1 内容确认无误后：设为只读锁定（推荐）

写入的最终内容测试通过、确认不会再改之后，在 NFC Tools（或同类 App）里找到 **"锁定标签 / Lock tag / 写保护"** 选项，执行锁定。

⚠️ **锁定不可逆**——锁定后这张卡片的 NDEF 内容永远不能再改写（但读取不受影响，网页依然正常打开）。
好处是不用担心以后被其他 NFC App 意外覆盖、格式化或清空。

**锁定前务必做到**：
- 内容（名字/日期/URL）已经反复确认无误
- 已经用真实手机实测过打开效果
- 网页内容（时间线/小纸条）不需要跟 URL 绑死——**只有 NDEF 里的 URL 被锁定，网页本身的内容永远可以照常更新**，这也是"标签只存链接，内容在云端"设计的意义所在
